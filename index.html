University Companion Mobile App

I implemented a university companion app on two different occasions using a different mobile framework each time. The app consists of a lecture timetable page where you can add, modify, and remove lectures. It also consists of a pomodoro timer page which can be used to allocate your time for studying. Finally, the app contains user authentication functionality via Google Firebase.

The first implementation of the app was developed using the Ionic framework. The second implementation of the app was developed with the React Native framework. Each of these implementations used Google Firebase for user authentication, and Cloud Firestore for persistent data storage.


I found that the implementation of the app was most difficult when using the React Native framework. I do believe this was solely due to my lack of experience with the framework combined with its steep learning curve. I do like how much control you have over the app though as the development process is very modular.

SCLoot Discord Bot

This is a Discord bot that I developed using the Java API wrapper called Discord4J. This Discord bot is primarily used for my Discord server called SCLoot which is dedicated to the game called StarBreak. The purpose of this server is to share what loot drops each person gets for the Star Cathedral. The server has over 400 members which makes it quite active as StarBreak has a relatively small player base.

The bot tallies up the loot submitted by the players in the server in preparation for the automated weekly/monthly reports, as shown below.

This is the bulk of the Discord Bot. It really brings the community together as each person shares their loot, which otherwise becomes a mystery. There are some additional features that I have also integrated into the bot:

  • Moderation tools - (kicking/banning/warning) users.
  • Auto-updating ranked role system based on how many submissions a user has.
  • User authentication via messaging the bot upon entering the server, which checks whether the given username is a valid StarBreak username or not.
  • Restricting reaction emojis to only emojis that are contained in each submission.
I dealt a lot with Java Dates during the coding process. I am grateful that it was designed in a functional style (unlike JavaScript Dates) as it ensured there were no potential cross-referencing problems.

The Wandering Alien

The Wandering Alien is a 2D puzzle platformer game that is currently being developed using the Godot game engine. Your goal in the game is to find the exit at each level, collecting as many stars as you can along the way.

I developed scripts using the engine's scripting language called GDscript (which is similar to Python) to control the logic of each entity in the game. I have also implemented persistent game saving and loading functionality via local device storage. The game is currently still in development.

Dungeon Raider

Dungeon Raider is a medieval-fantasy themed 2D dungeon crawler game. Your goal is to fight monsters, upgrade your weapon, and reach the end of the dungeon. You are given a sword which you must use to kill the monsters to progress through the entire dungeon.

This game was developed in a team of five members for a group project for SWEN222 (back in early 2017). The game was developed using vanilla Java without the use of any external frameworks or libraries. Different components of the game were designated to each team member, and I ended up being responsible for the map rendering, loading, and saving. The map utilised a tile system which made saving and loading much more simple.
